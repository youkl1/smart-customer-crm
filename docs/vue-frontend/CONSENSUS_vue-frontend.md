# 需求共识文档 - Vue前端代码生成

## 明确的业务目标与需求描述
- **业务目标**：构建一套完整的智客通CRM前端系统，实现企业客户全生命周期管理，提升销售效率和管理水平
- **核心需求**：
  1. **端选择与统一登录**：提供统一的登录入口，支持WEB端和移动端切换
  2. **移动端小程序**：实现销售人员日常操作所需的核心功能，包括客户管理、商机管理、任务管理等
  3. **WEB端后台管理**：提供完整的企业级管理功能，包括数据报表、审批流程、系统配置等
  4. **技术架构**：采用Vue 3 + Composition API，使用Vant UI和Element Plus实现不同端的界面

## 用户场景与核心流程
### 1. 端选择与登录流程
- **场景**：用户打开系统，选择使用WEB端或移动端
- **流程**：
  1. 用户访问系统首页，看到端选择界面
  2. 用户选择对应端（WEB端或移动端）
  3. 跳转至对应端的登录页面
  4. 用户输入账号密码，完成登录
  5. 系统验证登录信息，存储登录状态
  6. 跳转至对应端的首页

### 2. 移动端小程序使用流程
- **场景**：销售人员在外拜访客户，需要管理客户信息和商机
- **流程**：
  1. 销售人员登录移动端小程序
  2. 在客户列表中查看客户信息
  3. 点击客户进入详情页，查看联系人、跟进记录等
  4. 新增客户或商机，录入相关信息
  5. 在任务中心查看待办任务，完成任务标记
  6. 在报表看板查看个人业绩

### 3. WEB端后台管理使用流程
- **场景**：销售经理或系统管理员在办公室使用电脑管理系统
- **流程**：
  1. 登录WEB端后台管理
  2. 在工作台查看数据概览
  3. 进入客户管理模块，进行客户分配、合并等操作
  4. 进入商机管理模块，查看商机漏斗，推进商机阶段
  5. 进入合同管理模块，审批合同
  6. 进入系统设置模块，配置组织架构和权限

## 可量化的验收标准
1. **功能完整性**：
   - 端选择与登录模块功能完整，支持端切换和登录验证
   - 移动端小程序所有功能模块实现，包括客户、联系人、商机、任务、报表、消息、个人中心
   - WEB端后台管理所有功能模块实现，包括工作台、客户、联系人、商机、合同、产品、订单、回款、报表、系统设置、个人中心

2. **技术实现**：
   - 使用Vue 3 + Composition API（`<script setup>`语法）
   - 移动端使用Vant UI，WEB端使用Element Plus
   - 路由管理使用vue-router 4.x，状态管理使用Pinia
   - 图表库使用ECharts 5.x
   - API请求统一封装，提供模拟数据

3. **界面设计**：
   - 端选择页全屏居中布局，按钮宽度自适应
   - 移动端固定宽度375px，水平居中，高度自适应
   - WEB端经典后台布局，左侧导航栏 + 顶部Header + 右侧内容区
   - 界面美观、易用，符合现代前端设计标准

4. **性能与体验**：
   - 页面加载速度快，响应及时
   - 移动端支持下拉刷新和上拉加载
   - WEB端表格支持排序、筛选、分页
   - 路由切换流畅，无明显卡顿

5. **代码质量**：
   - 代码结构清晰，符合项目规范
   - 变量、函数命名语义化
   - 关键逻辑有注释
   - 代码可维护性高

## 产品与技术实现方案框架
### 1. 产品方案
- **端选择页**：展示系统Logo和名称，提供WEB端和移动端选择按钮
- **登录页**：账号密码输入、验证码、记住密码、自动登录
- **移动端**：底部导航栏（客户、商机、任务、我的），顶部标题栏
- **WEB端**：左侧固定导航栏，顶部用户信息和操作栏，右侧内容区

### 2. 技术方案
- **前端框架**：Vue 3 + Composition API
- **UI组件**：Vant UI（移动端）、Element Plus（WEB端）
- **路由管理**：vue-router 4.x，按端类型拆分路由
- **状态管理**：Pinia，管理用户状态和全局配置
- **图表库**：ECharts 5.x，用于数据可视化
- **API封装**：统一的request.js，模拟请求和响应

   - 代码结构必须按照规定的目录结构组织

2. **业务约束**：
