# 需求对齐文档 - Vue前端代码生成

## 项目与任务特性规范
- **项目名称**：智客通CRM（SmartCustomer CRM）
- **任务名称**：Vue前端代码生成
- **任务类型**：前端开发
- **技术栈**：Vue 3 + Composition API、Vant UI（移动端）、Element Plus（WEB端）、vue-router 4.x、Pinia、ECharts 5.x

## 原始需求原文
生成一套完整的Vue前端代码，包含「端选择-统一登录」入口、**移动端小程序**（固定宽度375px，高度自适应）和**WEB端后台管理**两大业务端，实现企业客户全生命周期管理功能。

## 业务背景与目标
- **业务背景**：企业需要一套完整的CRM系统，实现客户全生命周期管理，包括客户信息管理、商机跟进、合同管理等功能
- **核心目标**：
  1. 提供统一的端选择和登录入口
  2. 实现移动端小程序，方便销售人员随时随地管理客户和商机
  3. 实现WEB端后台管理，提供完整的企业级管理功能
  4. 确保系统的易用性、稳定性和安全性

## 任务边界确认
### 明确做什么
- 生成完整的Vue前端项目结构
- 实现端选择和统一登录模块
- 实现移动端小程序功能（客户管理、联系人管理、商机管理、任务中心、报表看板、消息通知、个人中心）
- 实现WEB端后台管理功能（工作台、客户管理、联系人管理、商机管理、合同管理、产品管理、订单管理、回款管理、报表中心、系统设置、个人中心）
- 实现路由管理、状态管理、API请求封装等核心功能

### 明确不做什么
- 不实现后端API接口，仅提供模拟数据
- 不涉及数据库设计和实现
- 不实现真实的小程序打包和发布
- 不涉及服务器部署和运维

## 需求理解说明
### 技术栈要求
- **核心框架**：Vue 3 + Composition API（`<script setup>`语法）
- **UI组件库**：
  - 移动端：Vant UI（最新版）
  - WEB端：Element Plus（最新版）
- **路由管理**：vue-router 4.x，区分「端选择路由」「移动端路由」「WEB端路由」
- **状态管理**：Pinia，存储登录状态、用户信息、当前端类型、全局配置
- **图表库**：ECharts 5.x（用于WEB端报表、移动端简易看板）
- **HTTP请求**：统一封装`request.js`（模拟请求，预留真实接口对接位置）

### 功能模块理解
1. **端选择与统一登录模块**：
   - 端选择首页：展示系统Logo、名称，提供端选择按钮
   - 统一登录页：账号密码登录，包含验证码、记住密码等功能
   - 权限与路由守卫：实现未登录跳转、端类型守卫、Token过期处理

2. **移动端小程序**：
   - 客户管理：客户列表、详情、新增/编辑、公海池
   - 联系人管理：联系人列表、新增/编辑/删除
   - 商机管理：商机列表、详情、新增/编辑、阶段推进
   - 任务中心：待办/已办任务、新增/编辑、完成标记
   - 报表看板：个人业绩概览、跟进趋势
   - 消息通知：系统消息、审批消息
   - 个人中心：个人信息、修改密码、退出登录

3. **WEB端后台管理**：
   - 工作台：数据概览、销售漏斗、业绩趋势
   - 客户管理：客户列表、详情、批量操作、导入/导出
   - 联系人管理：联系人列表、新增/编辑/删除、导入/导出
   - 商机管理：商机列表、详情、阶段推进、赢单/输单
   - 合同管理：合同列表、详情、审批流程、归档/作废
   - 产品管理：产品分类、列表、新增/编辑/删除
   - 订单管理：订单列表、新建/编辑/取消、发货操作
   - 回款管理：回款记录、新建回款、审批、回款计划
   - 报表中心：客户报表、销售报表、财务报表、自定义报表
   - 系统设置：组织架构、角色权限、字典管理、系统日志
   - 个人中心：个人信息、修改密码、退出登录

### 页面布局与适配理解
- **端选择页**：全屏居中布局，按钮宽度自适应屏幕
- **移动端**：固定宽度375px，水平居中，高度100vh自适应，使用Vant的viewport适配方案
- **WEB端**：经典后台布局，左侧固定导航栏 + 顶部Header + 右侧内容区，支持响应式

## 歧义与疑问澄清清单
1. **关于移动端小程序**：
   - 问题：是否需要实现真实的小程序打包？
   - 决策：仅实现前端代码，不涉及小程序打包和发布

2. **关于API接口**：
   - 问题：是否需要对接真实后端API？
   - 决策：仅提供模拟数据，预留真实接口对接位置

3. **关于权限管理**：
   - 问题：是否需要实现完整的权限控制逻辑？
   - 决策：实现基本的路由守卫和端类型控制，权限管理功能在UI层面展示

4. **关于数据存储**：
   - 问题：是否需要实现本地存储？
   - 决策：使用Pinia进行状态管理，登录状态可考虑使用localStorage持久化

5. **关于响应式设计**：
   - 问题：WEB端是否需要支持移动端设备？
   - 决策：WEB端主要针对桌面设备，最小宽度1366px，移动端功能由小程序端提供
